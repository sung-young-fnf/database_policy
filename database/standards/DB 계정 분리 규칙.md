DB 계정을 서비스(앱)/개발자/읽기전용 3종으로 분리하고, 서비스 계정에서 DDL 권한을 제거하는 규칙을 정의한다. 보안 진단 지적(서비스/개발자 계정 미분리, 서비스계정 DDL 보유)에 대한 대응 정책이다. 계정 이름 규칙은 [[DB 계정 네이밍 규칙]], PostgreSQL Owner 관리는 [[PostgreSQL Owner 관리 규칙]] 참조.

## 계정 3종 분리

| 계정 유형 | 용도 | 권한 범위 | 발급 대상 |
|----------|------|----------|----------|
| **서비스 계정** | 애플리케이션(WAS) 접속 | **DML 전용** (기본) | 애플리케이션 |
| **개발자 계정** | DDL 실행, 개발자 사용 | DML + DDL | 개발자 |
| **읽기전용 계정** | 모니터링, 조회 전용 | SELECT만 | 모니터링 도구 |

## DBMS별 계층 구조

| DBMS | 계층 구조 | Schema와 User |
|------|----------|---------------|
| **PostgreSQL** | Instance → Database → Schema → Objects | Schema ≠ User (별개) |
| **Oracle** | CDB(Instance) → PDB(Database) → Schema → Objects | **Schema = User (동일)** |

> Oracle은 Schema = User이므로 서비스계정에서 DDL을 완전히 분리하기 어렵다. 버전별 대응 방안은 [[Oracle DB 계정 생성 런북]] 참조.

## DBMS별 계정 분리 방식

| 항목 | Oracle 19c (★4안) | Oracle 23ai (★5안) | PostgreSQL |
|------|-------------------|---------------------|------------|
| **서비스 계정** | 기존 스키마 계정 유지 | 기존 스키마 계정 유지 | `_svc` 신규 생성 (DML만) |
| **개발자 계정** | `_OPER` 1개 (DB 전체) | `_OPER` 스키마별 별도 | `_oper` (공유 또는 개인) |
| **DDL 부여** | `DDL_DML_ROLE` (Global ANY) | PL/SQL (ON SCHEMA) | `object_owner_role` (NOLOGIN) |
| **앱 설정 변경** | 불필요 | 불필요 | 필요 (접속 계정 `_svc`로 변경) |

## 서비스 계정 DDL 제거 사유

| # | 사유 | 설명 |
|---|------|------|
| 1 | 서비스 운영에 DDL 불필요 | 앱(WAS)은 데이터 조작(DML)만 수행 |
| 2 | 해킹 시 피해 범위 축소 | 서비스계정 탈취 시 DDL 가능하면 테이블 삭제/구조 변경으로 서비스 마비. DDL 제거 시 피해를 데이터 레벨로 한정 |
| 3 | AI/LLM Agent 오작동 방지 | AI의 잘못된 Tool Use로 테이블 구조 변경/삭제 가능. DDL 제거 시 DML 범위로 제한 |
| 4 | Logical 복제 활용 | 서비스계정 DDL 제한 시 블루/그린 배포 등 Logical 복제 활용 가능 |

## 서비스 계정 예외: DDL 필요 시

기본 정책은 서비스 계정에 DML만 부여. 앱이 스키마 마이그레이션(ALTER TABLE 등)을 직접 수행하는 경우 예외로 DDL 권한 부여 가능.

| DBMS | 예외 처리 방법 |
|------|-------------|
| Oracle | 스키마 계정이 곧 서비스 계정이므로 DDL 자연 보유 (분리 불가) |
| PostgreSQL | `_svc`에 `dml_role` 대신 `object_owner_role` 부여 |

> 예외 요청 예시: "이번 서비스계정(앱)을 만들 때 DML role 말고 object_owner_role을 부여해줘"

## 금지 사항

| 금지 항목 | 사유 |
|-------|-----|
| 서비스/개발자 계정 공용 사용 | 보안 로깅/모니터링 불가, 권한 별도 제어 불가 |
| 서비스 계정에 DDL 권한 부여 (기본) | 해킹/AI 오작동 시 서비스 마비 위험 |
| Oracle 개인에게 전역 ANY 직접 부여 | 19c: DDL_DML_ROLE에 번들, 23ai: ON SCHEMA로 스코프 |
