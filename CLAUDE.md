# Database KB 문서 작성 가이드

## 목적

DB 계정 분리 정책 문서(01~03)를 Outline KB에 등록하기 위한 KB 문서를 작성한다.
AI가 사용자 질문에 대해 **답변/절차 안내**를 할 수 있도록, 문서 간 [[]] 참조로 탐색 경로를 설계한다.

## 작성할 문서 목록 (9개)

```
📂 Database (Platforms 하위, Cloud/Workplace/Domain과 동일 레벨)
│
├── Database Platform Index              ← IX-DB-001
├── 📂 Standards
│   ├── DB 계정 분리 규칙                ← ST-DB-001 (접속 계정 나누기 — Oracle/PG 공통)
│   ├── DB 계정 네이밍 규칙              ← ST-DB-002 (이름 짓는 법 — Oracle/PG 공통)
│   └── PostgreSQL Owner 관리 규칙       ← ST-DB-003 (소유권 계층 관리 — PG 전용)
├── 📂 Runbooks
│   ├── Oracle DB 계정 생성 런북          ← RB-DB-001
│   ├── PostgreSQL DB 계정 생성 런북      ← RB-DB-002
│   ├── DB 개발자 계정 운영 런북          ← RB-DB-003
│   └── DB 계정 정책 점검 런북            ← RB-DB-004
└── Database Service Dependencies        ← IX-DB-002
```

## [[]] 참조 작성 규칙

`[[문서명]]`은 AI가 문서 간 탐색할 수 있는 **경로 힌트**다. 단순 링크가 아니라 **"어떤 상황에서 이 문서를 봐야 하는지"** 조건을 함께 명시한다.

### 위치별 작성법

**1. 메타블록 `관련문서` — 항상 같이 봐야 할 문서**
```markdown
| 관련문서 | [[DB 계정 분리 규칙]], [[PostgreSQL DB 계정 생성 런북]] |
```

**2. 관련 문서 섹션 — 이유와 함께**
```markdown
* > 구현 런북: [[Oracle DB 계정 생성 런북]] — 이 규칙의 Oracle 실행 절차
* > 구현 런북: [[PostgreSQL DB 계정 생성 런북]] — 이 규칙의 PostgreSQL 실행 절차
* > 관련: [[DB 개발자 계정 운영 런북]] — 생성 후 개발자에게 안내할 사용법
* > 관련: [[DB 계정 정책 점검 런북]] — 생성 후 점검 절차
```

**3. 본문 중 조건부 분기 — "이 상황이면 이 문서를"**
```markdown
## 에스컬레이션 기준
| Owner 혼재 발견 시 | [[DB 계정 정책 점검 런북]] 의 Owner 점검 SQL 실행 | DBA |
| 개발자 계정이 필요한 경우 | [[DB 개발자 계정 운영 런북]] 참조하여 요청 안내 | 요청자 |
```

**4. Platform Index 라우팅 — 키워드 → 문서 매핑**
```markdown
| 계정 생성, Oracle, 스키마 | [[Oracle DB 계정 생성 런북]] | 🔴 |
| 계정 생성, PostgreSQL, Role | [[PostgreSQL DB 계정 생성 런북]] | 🔴 |
| 네이밍, _svc, _oper, Owner | [[DB 계정 분리 규칙]] | - |
```

### 연결 규칙

- **표준 ↔ 표준**: 서로 관련된 규칙이면 연결 (예: 네이밍 규칙에서 계정 유형 참조 시)
- **표준 → 런북**: `> 구현 런북: [[런북명]]` — 이 규칙을 실행하는 절차
- **런북 → 표준**: `> 관련: [[표준명]]` — 이 절차의 근거 규칙
- **런북 ↔ 런북**: 에스컬레이션/트러블슈팅에서 조건부 연결 — "이 상황이면 이 절차"
- **Index → 전체**: 빠른 라우팅 테이블에서 키워드별로 모든 표준/런북 연결
- **Dependencies ↔ Index/런북**: 서비스별 관련 문서 연결

## 작성 순서

1. **ST-DB-001** (계정 분리 규칙) — 계정 유형/권한 범위 기준
2. **ST-DB-002** (계정 네이밍 규칙) — Oracle/PG 네이밍 패턴
3. **ST-DB-003** (PostgreSQL Owner 관리 규칙) — Owner 3단 분리/Role 체인
4. **RB-DB-001** (Oracle 런북) — 01-dbuser.md의 Oracle 절차
5. **RB-DB-002** (PostgreSQL 런북) — 01-dbuser.md의 PostgreSQL 절차
6. **RB-DB-003** (개발자 운영 런북) — 02-dbuser_developer_guide.md
7. **RB-DB-004** (점검 런북) — 03-dbuser_policy_check.md
8. **IX-DB-002** (Dependencies) — 서비스 간 의존성
9. **IX-DB-001** (Platform Index) — 모든 문서가 완성된 후 라우팅 테이블 작성

## 공통 작성 규칙

- **첫 문단은 요약**으로 시작 — AI가 전체를 읽지 않아도 문서 성격을 판단
- **키워드에 동의어/약어 포함** — `DB 계정`, `database account`, `_svc`, `_oper` 등
- **코드 블록에 언어 명시** — `sql`, `bash`, `hcl`, `json`
- **테이블 기반** — 규칙, 절차, 비교는 테이블로
- **문서ID 형식 준수** — `ST-DB-001`, `RB-DB-001`, `IX-DB-001`
